# Stage 4: Content and Difficulty

## Goal
Make the game replayable and configurable.

## Scope
### 4.1 Image Sources
- Support preset images.
- Allow future extension for custom image upload.

### 4.2 Board Size Selection
- Support board sizes such as 3x3 and 4x4.
- Keep logic generic for future sizes.

### 4.3 Solvable Shuffle
- Shuffle into valid, solvable states.
- Avoid impossible board arrangements.

## Output of this stage
- The game can support multiple images and board sizes.

## Dependencies
- Stage 2 core gameplay.

## Notes
- Shuffle logic must preserve solvability.
